# Статический класс

## Определение

Статический класс (или класс со статическими членами) в ООП — это концепция, где данные и методы принадлежат самому классу, а не его отдельным экземплярам. Они доступны без создания объекта, инициализируются один раз и используются для хранения общих данных или вспомогательных функций (утилит)

_ _ _

## Реализация

``` python
class StaticClass:
    """
    Класс, который НЕ предполагает создание экземпляров.
    Все методы и атрибуты — статические (принадлежат классу, а не объекту).
    """
    
    # Статический атрибут (принадлежит классу)
    class_attribute = "значение по умолчанию"
    
    # Запрет создания объектов (опционально)
    def __new__(cls, *args, **kwargs):
        raise TypeError(f"Класс {cls.__name__} не предназначен для создания объектов")
    
    # Статический метод №1
    @staticmethod
    def static_method_one(param1, param2):
        """
        Статический метод — не имеет доступа к self или cls.
        Работает как обычная функция, но находится внутри класса.
        """
        result = param1 + param2
        return result
    
    # Статический метод №2
    @staticmethod
    def static_method_two(value):
        return f"Обработано значение: {value}"
    
    # Метод класса — имеет доступ к cls (самому классу)
    @classmethod
    def class_method(cls, new_value):
        """
        Метод класса — может изменять статические атрибуты класса.
        """
        cls.class_attribute = new_value
        return f"Атрибут класса изменён на: {new_value}"
    
    # Метод класса для получения значения
    @classmethod
    def get_class_attribute(cls):
        return cls.class_attribute

```

***Ключевые особенности Python при создании объектов***

| Особенность | Описание |
|-------------|----------|
| `@staticmethod` | Декоратор, превращающий метод в статический (нет доступа к `self` или `cls`) |
| `@classmethod` | Декоратор — метод получает класс (`cls`) вместо объекта, может менять состояние класса |
| Отсутствие `static` ключевого слова | В Python нет ключевого слова `static` — используются декораторы |
| Модуль как альтернатива | Лучшая практика в Python — использовать модуль вместо класса со статикой |
| Запрет создания объектов | Можно переопределить `__new__`, чтобы выбросить исключение |
| Статические атрибуты | Объявляются на уровне класса (вне методов), принадлежат классу, а не экземпляру |
